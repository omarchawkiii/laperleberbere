📊 STRUCTURE DE LA BASE DE DONNÉES
═══════════════════════════════════════════════════════════════════════════════

TABLE: disponibilites
├─ id (INT, PRIMARY KEY)
├─ date (DATE)
├─ heure (TIME)
├─ statut (ENUM: 'libre' / 'réservé')
└─ created_at (TIMESTAMP)

TABLE: rendezvous
├─ id (INT, PRIMARY KEY)
├─ nom (VARCHAR 100)
├─ prenom (VARCHAR 100)
├─ email (VARCHAR 150)
├─ telephone (VARCHAR 20)
├─ raison (TEXT, NULLABLE)
├─ disponibilite_id (INT, FOREIGN KEY)
├─ statut (ENUM: 'en attente' / 'validé')
└─ created_at (TIMESTAMP)
